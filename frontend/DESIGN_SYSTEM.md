# 前端设计系统

本前端以 `stitch_hybrid_batch_file_processor/` 的 Stitch 导出页面作为视觉来源，保留其 Windows 桌面工具软件风格：固定侧边栏、顶部工具栏、卡片列表、参数面板、日志面板、风险标签和执行状态反馈。

## 设计变量

- 颜色：集中定义在 `assets/css/app.css` 的 `:root`，沿用 Stitch 的 surface、primary、status、outline、log-bg 等色板。
- 字体：所有界面文字统一使用本地 PingFang 字体，字体文件来自 `app/run.py` 同级目录并复制到 `assets/fonts/`。
- 间距：侧边栏 240px，内容外边距 32px，栅格间距 20px，表单间距 16px，卡片内边距 20px。
- 圆角：基础控件 4px，卡片/面板/侧边栏选中态 8px，完整圆角只用于状态标签。
- 阴影：卡片和面板使用柔和一级阴影，弹层/强调层保留更强二级阴影变量。

## 可复用组件

组件函数集中在 `assets/js/components.js`，页面只在 `assets/js/app.js` 中组合组件和请求数据。

- `UI.shell`：应用布局、侧边栏导航、顶部标题栏。
- `UI.button`：主按钮、次按钮、危险按钮、幽灵按钮和图标按钮变体。
- `UI.card`：小工具和固定流程目录卡片。
- `UI.panel`：带标题、说明、操作区、内容区和底部操作栏的面板。
- `UI.field`：根据 schema 渲染输入框、下拉框、文本域和开关。
- `UI.metric`：首页统计卡片。
- `UI.tag`：风险等级与任务状态标签。
- `recordsTable`：首页和执行记录页共用的记录表格。

## 页面开发规则

- 开发页面时优先复用 `UI.*` 组件。
- 如果已有组件可以通过参数、变体或额外 class 扩展，应优先扩展。
- 只有现有组件无法表达新的交互时，才新增组件。
- 页面函数只负责取数、组织数据和组合组件，不在页面里重复堆叠样式。
- 执行页和详情页使用 `workbench-grid`；目录页使用 `grid`；首页使用 `dashboard-grid`。
- 远程 UI 资源不得直接引用；CSS、字体、图标、favicon、manifest 都从本地静态资源加载。

## 响应式规则

- 默认桌面布局为 240px 固定侧边栏 + 自适应内容区。
- 视口宽度小于 1200px 时，侧边栏折叠为图标模式，首页统计变为两列，执行/详情页面变为单列。
